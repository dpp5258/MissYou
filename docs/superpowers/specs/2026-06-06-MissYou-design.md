# MissYou 设计说明书

## 整体目标

搭建一个**思念量化展示网页**，底层技术复用 Streamlit + GitHub + Streamlit Cloud 免费架构。核心功能：

- 管理员线下出售思念 tokens，在网页后台手动增减余额
- 思念值每日自动衰减（衰减速度管理员可随时调整）
- 生成外网链接发给用户，输密码查看思念余额，不可修改
- 星空夜幕主题 🌙，情感化视觉设计

---

```
┌──────────────────────────────────────────┐
│             Streamlit Cloud               │
│                                           │
│  ┌──────────┐  ┌──────────────┐          │
│  │ 用户页面  │  │  管理员后台   │          │

│       └───────┬───────┘                   │

        ┌───────▼───────┐
```

- **用户页面**：输入用户密码 → 星空主题情感化页面展示思念余额
- **管理员后台**：输入管理密码 → 加减 tokens、调整每日衰减速度、查看操作记录
- **Google Sheets**：存储余额、衰减参数、操作日志，只有管理员账号可访问

## 三、Google Sheets 数据设计

一个 Google Sheet 文件，内含两个工作表：

### 工作表①：`账户状态`

| 列名 | 说明 | 示例值 |
|------|------|--------|
| `当前思念值` | 用户可见的 token 余额 | 8500 |
| `每日衰减量` | 每天自动减多少 token | 10 |
| `最后更新时间` | 上次触发衰减计算的日期 | 2026-06-06 |
| `思念起始日` | 思念从哪天开始算起 | 2025-08-15 |

### 工作表②：`操作记录`

| 列名 | 说明 | 示例值 |
|------|------|--------|
| `时间` | 操作时间戳 | 2026-06-06 14:30 |
| `操作类型` | 手动增加 / 手动扣除 / 调整衰减 / 自动衰减 | 手动增加 |
| `数值变化` | 带符号的数值变化 | +500 |
| `操作后余额` | 操作完成后的余额 | 9000 |
| `备注` | 管理员填写的原因 | 用户线下购买 |

### 自动衰减逻辑

每次用户或管理员打开页面时，程序自动执行：

1. 读取 `最后更新时间` 和 `每日衰减量`
2. 计算距今经过的天数
3. 从 `当前思念值` 中扣除 `天数 × 每日衰减量`
4. 更新 `当前思念值` 和 `最后更新时间`
5. 写入一条"自动衰减"操作记录

5. **注销**：退出管理员状态，回到密码门

## 五、安全与权限

| 层面 | 措施 |
|------|------|
| **用户权限** | 只有查询密码，看不到管理后台，无法修改数据 |
| **管理员权限** | 独立管理密码，与用户密码不同 |
| **Google Sheets** | 使用 Service Account 凭证文件，只有你的 Google 账号可访问原始数据 |
| **Streamlit Cloud** | 公开部署，密码保护内容，仓库代码公开但凭证不入库 |
| **凭证安全** | Google Service Account JSON 存为 Streamlit Secrets，不暴露在代码中 |

## 七、部署步骤概要

1. 创建 Google Cloud 项目，启用 Sheets API，生成 Service Account 凭证
2. 创建 Google Sheet，分享给 Service Account 邮箱
3. 本地编写 `app.py` 和 `requirements.txt`
4. 推送到 GitHub 公开仓库
5. Streamlit Cloud 关联仓库，配置 Secrets（凭证 JSON + 密码）
6. 部署，获取 `xxx.streamlit.app` 外网链接
7. 将链接和用户密码发给对方

**后续修改：**
- 管理员直接在网页后台操作，实时生效，无需重新部署
- 如修改代码逻辑，本地改完后推送到 GitHub，Streamlit 自动更新

## 八、扩展预留

- **多用户模式**：Google Sheet 增加"用户表"，侧边栏切换不同用户
- **思念等级**：根据余额触发不同视觉特效（如余额高时星星更多）
- **微信通知**：余额低于阈值时，通过企业微信或 Server 酱推送提醒
